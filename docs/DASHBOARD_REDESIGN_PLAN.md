# Plan: Dashboard redesign (Desktop App)

Дизайн-источник: `ai_studio_code (3).html`  
Scope: **только `DashboardPage.tsx`** — layout других страниц не трогаем, ждём отдельные дизайны.

---

## Принципиальное ограничение

Сайдбар в дизайне есть, но **пока мы не делаем общий `AppLayout`** — это сломает все остальные страницы.  
Вместо этого сайдбар рендерится прямо внутри `DashboardPage` как локальный компонент.  
Когда придут дизайны для остальных страниц — вынесем в `AppLayout` отдельным шагом.

---

# Plan: QuizEditorPage redesign

Дизайн-источник: `quiz-design.html`  
Важно: **сайдбар отсутствует** — редактор имеет собственную топ-панель.

## Принципиальное решение

Текущий редактор уже близок к дизайну структурно (левая панель вопросов + центральный редактор). Основная работа — визуальный редизайн существующих элементов, не реструктуризация логики.

## Задачи

### Шаг 1 — Топ-бар

Заменить текущий `<header>` с кнопками-emoji:

- Кнопка `ArrowLeft` (Phosphor) → `navigate('/dashboard')`
- Breadcrumb: «Мои квизы › [название квиза]» — название в `bg-slate-100 px-3 py-1 rounded-lg`
- Pill-статус рядом с breadcrumb: «● Черновик» / «● Опубликован»
- Справа: inline «☁ Сохранено» (иконка `CloudCheck`) + кнопка «Превью» + кнопка «Опубликовать» (primary)
- Убрать кнопку «× Удалить» из хедера

### Шаг 2 — Левая панель вопросов

Редизайн списка вопросов:

- Заголовок «Вопросы (N)» + иконка сортировки
- Активный вопрос: `bg-primary-50 border-primary-200 shadow-sm`
- Неактивный вопрос: drag-иконка появляется при hover (`opacity-0 group-hover:opacity-100`)
- Номер вопроса: квадратный бейдж `rounded-lg`
- Под текстом — тип вопроса с иконкой (маппинг иконок Phosphor для каждого типа)
- Кнопка «+ Добавить вопрос»: dashed border, при hover → `border-primary-400`
- Дропдаун типов вопросов: всплывает снизу вверх, `rounded-2xl shadow-floating`

### Шаг 3 — Центральная зона редактора

Визуальный редизайн полей:

- Поля ввода: `premium-input` стиль (белый фон, `rounded-2xl`, `border-slate-200`, при фокусе `border-primary-300 ring-4 ring-primary-50`)
- Кнопка «✨ Улучшить» → `btn-magic` стиль (градиентный фон `#F3E8FF → #FFE4E6`, текст `primary-600`)
- Карточки вариантов ответов: `answer-card` стиль — `border-2 border-transparent`, при `is-correct` → `border-emerald-500 bg-emerald-50`
- Таймер и очки — компактные selects/inputs с `premium-input` стилем

### Шаг 4 — Правая панель (настройки квиза)

В текущей реализации настройки квиза (название, предмет, класс, фон) могут быть в отдельной вкладке или правой колонке. По дизайну выделяется правая колонка `~320px`:

- Название квиза, предмет, класс, описание
- Блок генерации фона: миниатюра превью + кнопка «Сгенерировать фон»

## Что НЕ делаем

- Переписывание логики DnD — только визуал
- Новые типы вопросов — только редизайн существующих
- Правая панель настроек — если в текущем коде она в модальном окне, пока оставить там

# Plan: PreviewPage redesign

Дизайн-источник: `preview-design.html`  
Важно: **fullscreen без навигации** — самостоятельный режим просмотра.

## Принципиальное решение

Дизайн сильно отличается от текущего визуально, но логика (QuizEngine, состояния idle/playing/feedback/finished) остаётся той же. Переписываем только то, что рендерится — не логику.

## Задачи

### Шаг 1 — Фон и базовый layout

- Текущий: `radial-gradient` по `quiz.subject` — **оставить**, это правильное поведение
- Добавить `mesh-bg` — анимированный псевдоэлемент поверх градиента (медленное вращение, `animation: spin 20s linear infinite alternate`)
- Добавить тонкую noise-текстуру через SVG data URI (как в дизайне)
- `body`: `overflow: hidden`, `flex flex-col`, `h-screen`

### Шаг 2 — Топ-бар

Заменить текущий хедер:

- `glass-header`: `bg-slate-900/40 backdrop-blur-xl border-b border-white/5`
- Кнопка «Завершить» (`X`): `bg-white/5 border border-white/10 text-white/80`
- Прогресс-блок по центру: «Вопрос N / Total» + «100 очков» в `bg-black/30 rounded-xl`

### Шаг 3 — Круговой SVG-таймер

Заменить текущий `TimerBar` на круговой таймер:

- SVG 100×100, `r=45`, stroke `#10b981`, `strokeDasharray=283`
- `strokeDashoffset` обновляется через `useEffect` с `setInterval`
- При ≤5 секундах: цвет → `#ef4444`, число → `text-red-400`
- Обернуть в React-компонент `CircularTimer`

### Шаг 4 — Текст вопроса

- `text-3xl md:text-4xl lg:text-5xl font-black text-white`
- `text-wrap: balance` через кастомный CSS-класс
- `drop-shadow-2xl` для читаемости на тёмном фоне

### Шаг 5 — Тайлы ответов

Переделать текущие кнопки-тайлы:

- Структура: `[буква-колонка (80px)] [текст + иконка результата]`
- 4 цвета: ruby (`#e11d48`), sapphire (`#2563eb`), amber (`#d97706`), emerald (`#059669`)
- Блик при hover: `::after` pseudo-element скользит слева направо
- Состояние revealed: правильный → `scale(1.02) + white border + glow`; неправильные → `opacity-0.25 grayscale`; выбранный неверный → `shake animation + red border`
- Иконка результата (`Check`/`X`) внутри тайла — скрыта до reveal, `opacity-0 → opacity-100`

### Шаг 6 — Кнопка «Следующий вопрос»

- Floating кнопка: `fixed bottom-12 left-1/2 -translate-x-1/2`
- Появляется после reveal: `opacity-0 translate-y-8 → opacity-100 translate-y-0` (transition 500ms)
- Стиль: `rounded-full bg-white text-slate-900 font-black shadow-[0_10px_40px_rgba(255,255,255,0.2)] animate-pulse-glow`

## Что НЕ делаем

- Экран финального результата (leaderboard) — оставляем текущую логику, только базовый визуал
- Экран «Старт» — только если он уже есть в текущем коде (`state === 'idle'`)
