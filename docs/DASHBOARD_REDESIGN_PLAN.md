# Plan: Dashboard redesign (Desktop App)

Дизайн-источник: `ai_studio_code (3).html`  
Scope: **только `DashboardPage.tsx`** — layout других страниц не трогаем, ждём отдельные дизайны.

---

## Принципиальное ограничение

Сайдбар в дизайне есть, но **пока мы не делаем общий `AppLayout`** — это сломает все остальные страницы.  
Вместо этого сайдбар рендерится прямо внутри `DashboardPage` как локальный компонент.  
Когда придут дизайны для остальных страниц — вынесем в `AppLayout` отдельным шагом.

---

## Задачи

### Шаг 1 — Зависимости и конфигурация

- [ ] Установить `@phosphor-icons/react` в `apps/desktop`
- [ ] Подключить шрифт **Nunito** в `apps/desktop/index.html` (Google Fonts `<link>`)
- [ ] Обновить `tailwind.config.ts`:
  - Заменить `fontFamily.sans` → `['Nunito', 'system-ui', 'sans-serif']`
  - Добавить цвета `primary` (`violet-50/100/500/600/900`), `peach`, `mint`
  - Добавить `boxShadow`: `soft`, `card`, `input`

### Шаг 2 — Сайдбар (локальный компонент)

Новый файл: `apps/desktop/src/components/Sidebar.tsx`

Содержимое:

- Логотип «Квиз**ОК**» с иконкой `Lightning` из Phosphor
- Навигационные пункты: Главная, Мои квизы (активный), Ученики, Результаты
- Секция «Инструменты»: Библиотека, Шаблоны
- Футер: аватар + имя пользователя + должность (из `localStorage` или заглушка)
- Активный пункт: `bg-primary-50 text-primary-700`, неактивный: `text-slate-500 hover:bg-slate-50`

### Шаг 3 — Верхний хедер

Переписать `<header>` внутри `DashboardPage`:

- Убрать кнопки «Настройки / Демо / Обучение» из хедера (они уходят в сайдбар или footer)
- Добавить breadcrumb «Рабочее пространство › Мои квизы»
- Добавить поисковое поле (pill-форма, `rounded-full`, иконка `MagnifyingGlass`)
  - Состояние `searchQuery` в компоненте, фильтрует массив квизов
- Добавить кнопку-колокол уведомлений (заглушка, без бэкенда)

### Шаг 4 — AI-баннер (inline, не модальный)

Переосмыслить блок создания квиза:

- Вынести из модалки на главную как `magic-banner` — градиентный блок `lg:col-span-2`
- Иконка `MagicWand` из Phosphor, заголовок «Что создадим сегодня? ✨»
- Инпут с кнопкой «Придумать» прямо внутри баннера
- При сабмите: поведение как сейчас (`handleCreateAI`) — создать квиз и перейти в редактор
- **Модальное окно сохранить** — оно нужно для кнопки «+ Новый квиз» в хедере (пустой квиз)

### Шаг 5 — Блок «Быстрый старт»

Новая карточка `friendly-card` рядом с баннером (`lg:col-span-1`):

- Кнопка «Загрузить файл» — вызывает Tauri `dialog.open()`, функциональность уже есть
- Кнопка «Взять шаблон» — заглушка с `showToast('Скоро', 'info')`

### Шаг 6 — Фильтры и заголовок списка

Над сеткой карточек:

- Заголовок «Ваши материалы» + badge с кол-вом квизов
- Два pill-фильтра: «Все предметы» и «Любой класс»
  - Состояния `filterSubject` и `filterGrade`
  - `useMemo` для фильтрации + учёт `searchQuery`
  - Значения дропдаунов: уникальные `subject` и `gradeLevel` из загруженных квизов

### Шаг 7 — Карточки квизов (редизайн)

Заменить текущий вид карточки:

| Было                              | Станет                                                            |
| --------------------------------- | ----------------------------------------------------------------- |
| Градиентная шапка h-20            | Убрать — весь контент на белом                                    |
| Badge «Опубл./Черн.» вверху       | Цветная pill-метка предмета (emoji + текст) + класс-балка справа  |
| Постоянный footer с кнопками      | Кнопки появляются при hover (`opacity-0 group-hover:opacity-100`) |
| Кнопка «×» удалить — всегда видна | Убрать «×» из footer, перенести в hover-меню или kebab-иконку     |
| Статус через `Badge` компонент    | Строка «● Опубликован» / «● Черновик» внизу карточки              |

Мини-статистика: аватары сдавших учеников (если `leaderData` доступны — опционально в v2).
