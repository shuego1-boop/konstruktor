## 2. Дизайн-системы и библиотеки компонентов

### 2.1 shadcn/ui — ⭐ Рекомендация №1

|              |                                                                       |
| ------------ | --------------------------------------------------------------------- |
| Тип          | Copy-paste компоненты (не npm-зависимость)                            |
| Стек         | React + Tailwind CSS + Radix UI                                       |
| Компонентов  | 50+ (Dialog, Dropdown, Tabs, Toast, Tooltip, Sheet, Carousel, и т.д.) |
| Кастомизация | Полная — код ваш, стилизация через Tailwind                           |
| Доступность  | A11y из коробки (Radix UI)                                            |

**Почему подходит Konstruktor:**

- Уже используем Tailwind 4 + React — идеальная совместимость
- Компоненты копируются в `packages/ui` — не создают внешнюю зависимость
- Можно постепенно добавлять нужные компоненты (Dialog, Toast, Skeleton, Tabs)
- Поддерживает dark mode, анимации, composable API

**Что стоит взять из shadcn/ui:**

| -------------------- | ----------------------------------------------- |
| Dialog / AlertDialog | Подтверждение действий, превью вопросов         |

| Dropdown Menu        | Контекстные меню в конструкторе                 |
| Tabs                 | Навигация в CRM (аналитика, ученики, настройки) |

| Tooltip              | Подсказки к кнопкам                             |
| Sheet                | Мобильные боковые панели                        |

## 3. Анимационные инструменты

### 3.1 Framer Motion / Motion (уже установлен)

**Текущий статус:** установлен во всех пакетах, но почти не используется.

**Ключевые возможности, которые можно задействовать:**

| Возможность                       | Описание                        | Где применить                                 |
| --------------------------------- | ------------------------------- | --------------------------------------------- |
| `AnimatePresence`                 | Анимация появления/исчезновения | Переходы между вопросами, модалки             |
| `layout`                          | Автоматические layout-анимации  | Перестановка карточек вопросов в конструкторе |
| Variants + `staggerChildren`      | Каскадные анимации              | Появление вариантов ответа один за другим     |
| `whileHover` / `whileTap`         | Микро-интеракции                | Кнопки, карточки ответов                      |
| `useScroll` + `useTransform`      | Scroll-привязанные анимации     | Прогресс чтения, параллакс в CRM              |
| Spring physics                    | Физичные пружинные анимации     | Всё — кнопки, карточки, модалки               |
| Gestures (drag, pan)              | Жестовые анимации               | Drag-and-drop вопросов, swipe ответов         |
| `useMotionValue` + `useTransform` | Динамические значения           | Связь прогресса с цветом, размером            |

### 3.2 Lottie (lottie-react)

|                 |                                              |
| --------------- | -------------------------------------------- |
| Тип             | JSON-анимации из After Effects / LottieFiles |
| Размер          | Несколько КБ на анимацию                     |
| Качество        | Векторные, масштабируемые                    |
| Интерактивность | Привязка к скроллу, курсору, событиям        |

**Почему стоит добавить:**

- Тысячи бесплатных анимаций на [LottieFiles](https://lottiefiles.com)
- Идеально для иллюстративных анимаций: звёзды, трофеи, огонь, конфетти, галочки
- Малый размер (~5-50 КБ вместо GIF/видео)
- Работает в Capacitor (Android) без проблем

**Где применить:**

| Анимация                | Сцена                             |
| ----------------------- | --------------------------------- |
| ✅ Галочка              | Правильный ответ                  |
| ❌ Крестик с тряской    | Неправильный ответ                |
| 🏆 Трофей               | Завершение квиза, рекорд          |
| 🔥 Огонь                | Streak (серия правильных ответов) |
| ⭐ Звёзды               | Набор нового уровня               |
| 🎉 Конфетти / фейерверк | Максимальный балл                 |
| ⏳ Песочные часы        | Время на вопрос истекает          |
| 📊 Рост графика         | Аналитика в CRM                   |
| 💡 Лампочка             | Подсказка                         |
| 🚀 Ракета               | Быстрый ответ                     |

### 3.3 Rive

|          |                                          |
| -------- | ---------------------------------------- |
| Тип      | Интерактивные анимации со стейт-машинами |
| Runtime  | `@rive-app/react-canvas`                 |
| Редактор | rive.app (бесплатный для начала)         |

**Основное отличие от Lottie:**

- **Интерактивные стейт-машины** — анимация реагирует на ввод (клики, значения, триггеры)
- Идеально для персонажей-маскотов, геймифицированных элементов

**Где применить:**

- Маскот-персонаж, который реагирует на правильные/неправильные ответы
- Интерактивный прогресс-бар в виде персонажа, бегущего к финишу
- Анимированные переходы между уровнями сложности

**Стоимость:** бесплатно для open-source и неком. проектов; платно для команд.

### 3.4 tw-animate-css (Tailwind CSS v4)

|             |                                     |
| ----------- | ----------------------------------- |
| Тип         | Плагин Tailwind v4 для CSS-анимаций |
| Компонентов | 90+ утилитарных анимаций            |
| Benchmark   | 91.95 (лучший результат среди всех) |
| Zero JS     | Чистый CSS, нулевой overhead        |

**Полезно для:** простых анимаций появления, исчезновения, bounce, shake — без подключения JS-библиотек.

### 3.5 AutoAnimate

|        |                                                                   |
| ------ | ----------------------------------------------------------------- |
| Тип    | Zero-config автоматические анимации                               |
| Размер | ~2 КБ                                                             |
| Магия  | Одна строка кода добавляет анимации к спискам, условным элементам |

```tsx
import { useAutoAnimate } from "@formkit/auto-animate/react";

function QuestionList() {
  const [parent] = useAutoAnimate();
  return (
    <div ref={parent}>
      {questions.map((q) => (
        <QuestionCard key={q.id} />
      ))}
    </div>
  );
}
```

**Идеально для:** списков вопросов в конструкторе, фильтрации результатов в CRM.

## 5. Стратегия "вау-эффекта"

### 5.1 Для учеников (Player)

Цель: сделать прохождение квиза **захватывающим, как игру**.

#### Переходы между вопросами

```
Вопрос уходит → slide-out влево с fade
Новый вопрос → slide-in справа с spring bounce
Варианты ответа → stagger появление (0.1s между каждым)
```

**Инструмент:** Framer Motion `AnimatePresence` + variants

#### Обратная связь на ответ

```
Правильно:
  → Карточка вспыхивает зелёным (scale 1.05 → 1.0)
  → Lottie-анимация галочки
  → Тактильный feedback (Capacitor Haptics)
  → Звук "ding" (howler)
  → +100 очков с CountingNumber анимацией

Неправильно:
  → Карточка трясётся (shake animation)
  → Lottie-анимация крестика
  → Вибрация (Capacitor Haptics)
  → Правильный ответ подсвечивается с задержкой 0.5s
```

#### Streak-система

```
3 подряд → Badge "STREAK!" с bounce-in + звук
5 подряд → Badge "ON FIRE!" + Lottie 🔥 + экран слегка пульсирует
10 подряд → Badge "LEGENDARY!" + Lottie 🌟 + конфетти (canvas-confetti)
```

#### Таймер

```
> 50% времени → Спокойная зелёная полоса
30-50% → Жёлтая полоса + лёгкая пульсация
< 30% → Красная полоса + тряска + пульсация ускоряется
< 10% → Красная полоса мигает + звук тикания
```

**Инструмент:** Framer Motion `useMotionValue` + `useTransform` для связи progress → цвет/scale

#### Завершение квиза

```
→ Экран затемняется
→ Карточка результата "влетает" снизу (spring)
→ Счёт считается от 0 до финального (SlidingNumber)
→ Звёзды/рейтинг появляются с stagger
→ При >90% — конфетти + Lottie трофея
→ При <50% — мотивирующая анимация "В следующий раз получится!"
```

#### Лидерборд

```
→ Строки появляются с stagger сверху вниз
→ Текущий ученик подсвечивается пульсацией
→ При попадании в топ-3 — особая анимация (корона, медаль)
```

### 5.2 Для учителей (Desktop — Конструктор)

Цель: сделать создание квизов **приятным и интуитивным**.

#### Drag & Drop вопросов

```
→ Карточка "поднимается" (scale 1.02, shadow-floating) при захвате
→ Другие карточки плавно раздвигаются (layout animation)
→ Карточка "садится" на место с spring bounce
→ Placeholder мигает приглашающе
```

**Инструмент:** dnd-kit + Framer Motion `layout`

#### Добавление вопроса

```
→ Новая карточка "разворачивается" от нулевой высоты
→ Фокус плавно перемещается на поле ввода
→ Номер вопроса считается с анимацией
```

#### Сохранение / публикация

```
→ Кнопка "Сохранить" → ripple-эффект
→ Toast "Сохранено!" появляется с slide-in сверху
→ Кнопка "Опубликовать" → конфетти + Toast "Квиз доступен ученикам!"
```

#### Превью вопроса

```
→ При наведении на карточку вопроса — tooltip с миниатюрой
→ Клик → модалка с анимацией scale (0.95 → 1.0) + backdrop fade
```

#### Пустое состояние

```
→ Lottie-анимация "конструктора" или "пустой коробки"
→ Текст "Добавьте первый вопрос" с мягкой пульсацией на кнопке "+"
```

### 5.3 Для учителей (CRM — Аналитика)

Цель: сделать данные **живыми и понятными**.

#### Дашборд

```
→ Числа "считаются" от 0 до значения (CountingNumber / SlidingNumber)
→ Графики рисуются с анимацией (Recharts уже поддерживает)
→ Карточки статистики появляются с stagger
```

#### Детали ученика

```
→ Переход между страницами — shared layout animation
→ Прогресс-бар ученика анимируется при загрузке
→ Оценки появляются с color-coded fade-in
```

#### Пустые состояния

```
→ Если нет результатов — Lottie "поиск" / "пусто"
→ Если нет учеников — приглашающая анимация с QR-кодом
```

## 8. Доступность и производительность

### Обязательные правила

1. **`prefers-reduced-motion`** — все анимации должны отключаться:

   ```tsx
   const prefersReducedMotion = window.matchMedia(
     "(prefers-reduced-motion: reduce)",
   ).matches;
   // Framer Motion: <motion.div transition={prefersReducedMotion ? { duration: 0 } : spring} />
   ```

2. **GPU-ускорение** — анимировать только `transform` и `opacity` для 60fps:

   ```
   ✅ transform (translateX, scale, rotate)
   ✅ opacity
   ❌ width, height, top, left, margin, padding
   ```

3. **Бюджет анимаций:**
   - Максимум 3 одновременных анимации на экране
   - Lottie-файлы < 100 КБ каждый
   - Общий размер всех Lottie < 500 КБ

4. **Ленивая загрузка:**
   - Lottie-анимации загружать по требованию (`React.lazy` / `import()`)
   - Rive-файлы (если добавим) — только при входе на соответствующий экран

5. **Touch-устройства:**
   - Минимальный touch-target: 44×44px
   - Анимации hover не используются на touch — заменять на tap
   - Haptics (вибрация) только для ключевых моментов (правильно/неправильно)
