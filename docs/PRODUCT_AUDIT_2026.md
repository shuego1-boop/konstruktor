## 2. TOP-10 БЛОКЕРОВ ПОКУПКИ

### B1. Нет лендинга / сайта продукта [P0 · High]

**Стоп-фактор:** ЛПР гуглит «КвизОК» — находит ноль. Нет страницы, где можно понять продукт за 30 секунд.  
**Как исправить:** Развернуть лендинг на `kvizok.ru` / `kvizok.app`. Структура — раздел 4 этого аудита. Реализуемо за 3–5 дней на том же стеке (Vite + React + Tailwind). Разместить на DO App Platform как static site.

### B2. Нет тарифных планов и биллинга [P0 · High]

**Стоп-фактор:** «Сколько стоит?» — вопрос №1 от завуча. Если нет цены, нет покупки.  
**Как исправить:** Определить 3 тарифа (Бесплатный / Школа / Сеть). Добавить страницу `/pricing` в CRM и на лендинг. MVP биллинга: ЮKassa или CloudPayments, без автоматизации — по счёту. Stripe для экспорта.

### B3. Нет юридической основы и 152-ФЗ [P0 · High]

**Стоп-фактор:** Школа — бюджетное учреждение. Без политики обработки ПДн, без согласия родителей нельзя обрабатывать данные учеников (ФИО + результаты = персональные данные).  
**Как исправить:**

- Написать Политику конфиденциальности и Пользовательское соглашение.
- Добавить чекбокс согласия при регистрации.
- В Player: собирать только имя (первое), без фамилии, без email.
- Уведомление об обработке ПДн на экране ввода имени.
- Хранение данных на серверах в РФ (DO Frankfurt → Selectel/Yandex Cloud).

### B4. Нет ролевой модели для учреждения [P1 · High]

**Стоп-фактор:** Завуч хочет видеть аналитику по всем учителям. Директор — по всей школе. Сейчас CRM показывает только данные одного учителя.  
**Как исправить:**

- Добавить роли: `teacher`, `head_teacher` (завуч), `admin` (директор).
- Завуч видит квизы + результаты всех учителей своей организации.
- Директор видит сводку по организации + экспорт.
- Реализация: расширить `users.role` enum, добавить middleware фильтрации.

### B5. Нет демо-режима / триала без установки [P0 · High]

**Стоп-фактор:** Учитель не хочет устанавливать EXE до того, как увидит продукт. Завуч тем более.  
**Как исправить:**

- Веб-версия конструктора (iframe или отдельная страница) с предустановленным квизом.
- Кнопка «Попробовать бесплатно» на лендинге → регистрация в CRM → создание квиза в браузере.
- Player: веб-версия уже работает (Capacitor = обычный React). Развернуть на `play.kvizok.app`.

### B6. Нет PDF/КП для ЛПР [P0 · High]

**Стоп-фактор:** Завуч/директор принимает решение не в браузере, а на совещании. Нужен документ, который можно распечатать и показать.  
**Как исправить:**

- 2-страничный PDF: проблема → решение → тарифы → ROI → контакты.
- 5-слайдов PPTX для быстрой презентации на педсовете.
- Брендированный шаблон с логотипом КвизОК.

### B7. Нет social proof (отзывы, кейсы, цифры) [P1 · High]

**Стоп-фактор:** «Кто этим пользуется? А мы будем первые?» — школы крайне консервативны.  
**Как исправить:**

- Провести 3–5 бесплатных пилотов в школах.
- Собрать видео-отзывы (30 сек, телефон, неформат — это нормально).
- Цифры: «127 квизов создано», «1400 учеников прошли», «96% учителей вернулись».
- Разместить на лендинге + в КП.

### B8. Нет онбординга в CRM [P1 · Medium]

**Стоп-фактор:** Учитель зарегистрировался → видит пустой дашборд. Не понимает, что делать дальше.  
**Как исправить:**

- Empty state: «Создайте первый квиз» с пошаговой инструкцией.
- Видео-тур (2 минуты) на дашборде.
- Подсказки (tooltips) на первых 3 действиях.

### B9. Непоследовательный брендинг [P1 · Medium]

**Стоп-фактор:** В коде: «Konstruktor», «КвизОК», «Конструктор», «Konstruktor CRM», «Konstruktor Player». Для русскоязычной аудитории это путаница.  
**Как исправить:**

- Единое название: **КвизОК** (везде, на русском).
- Подзаголовок: «Платформа квизов для школ».
- HTML title: «КвизОК» (desktop), «КвизОК Плеер» (player), «КвизОК — Аналитика» (CRM).
- Лого уже интегрировано — теперь привести тексты.

### B10. Нет интеграции с экосистемой школы [P2 · Medium]

**Стоп-фактор:** IT-директор спросит: «Работает ли с нашим электронным журналом?»  
**Как исправить (после релиза):**

- API для экспорта результатов в формате CSV/XLSX (уже есть CSV).
- Интеграция с Дневник.ру API (post-MVP).
- SCORM-пакет для LMS (post-MVP).
- Плагин для Google Classroom (post-MVP).

## 3. TOP-10 РЫЧАГОВ РОСТА КОНВЕРСИИ

### G1. «Бесплатно навсегда» для 1 учителя [Учитель · P0]

**Выгода:** Учитель пробует без страха. Школа начинает с 1 энтузиаста.  
**Реализация:** Бесплатный тариф: 5 квизов, 30 учеников, базовая аналитика. Без ограничения по времени.

### G2. Готовые квизы по ФГОС [Учитель · P0]

**Выгода:** Учитель не создаёт, а берёт готовое. Экономия 2–3 часов на первом квизе.  
**Реализация:** Шаблоны уже есть (7 штук по предметам). Расширить до 20–30 по классам (5–11). Маркировать «по ФГОС 2024». Добавить в каталог на лендинге.

### G3. Отчёт для завуча в 1 клик [Администрация · P0]

**Выгода:** Завуч видит: «класс 6Б — средний балл 72%, слабые темы: дроби, площадь». Без ручного сбора.  
**Реализация:** Кнопка «Скачать отчёт» в CRM → PDF/XLSX с сводкой по классу/предмету/учителю. Автоматическая генерация инсайтов (уже есть в коде).

### G4. Калькулятор ROI на лендинге [Бизнес · P1]

**Выгода:** Директор видит: «3 часа экономии на учителя в неделю × 20 учителей × 40 недель = 2400 часов/год».  
**Реализация:** Интерактивный калькулятор: ввод кол-ва учителей → экономия часов → денежный эквивалент.

### G5. Мгновенный старт: «Квиз за 60 секунд» [Учитель · P0]

**Выгода:** Демонстрация простоты. AI уже умеет генерировать квиз по теме + классу.  
**Реализация:** На лендинге: поле ввода темы → AI генерирует квиз → предпросмотр → «Зарегистрируйтесь, чтобы сохранить». Максимально короткий путь к aha-момент.

### G6. QR-код для запуска квиза [Учитель · P1]

**Выгода:** Учитель показывает QR на доске → ученики сканируют → квиз открывается.  
**Реализация:** Генерация QR из shareCode квиза. Ссылка ведёт на `play.kvizok.app/?code=ABC-123`. Плеер открывается в браузере.

### G7. Живая доска результатов (Live Dashboard) [Учитель · P1]

**Выгода:** Учитель видит в реальном времени, кто проходит квиз, какой балл. Как Kahoot, но офлайн-совместимый.  
**Реализация:** WebSocket в API (Hono + Bun нативно поддерживает). Экран `/live/:quizId` в CRM. Обновление при синхронизации сессий.

### G8. Рейтинг класса за четверть [Администрация · P1]

**Выгода:** Завуч видит динамику: «6Б улучшил средний балл на 12% за четверть».  
**Реализация:** Агрегация результатов по периодам. Новая вкладка в CRM: «Динамика» с графиками по неделям/месяцам. Recharts уже подключен.

### G9. Белая этикетка (White Label) [Бизнес/Сеть · P2]

**Выгода:** Сеть школ или EdTech-компания хочет свой бренд.  
**Реализация:** Уже есть кастомизация темы (цвета, шрифты). Добавить: загрузку логотипа организации, кастомный домен (play.myschool.ru).

### G10. Геймификация: достижения + уровни [Учитель/Ученик · P2]

**Выгода:** Ученики возвращаются. Engagement растёт. Учитель видит мотивацию.  
**Реализация:** Achievements уже есть в коде (🌟 Perfect, ⚡ Speed Demon, 🔥 Consistent). Расширить: уровни (Новичок → Знаток → Мастер), недельные челленджи, badge wall.

## 5. ПОЗИЦИОНИРОВАНИЕ И ОФФЕР

### 5.1. Value Proposition (3 варианта)

**VP1 (для учителя):**

> КвизОК — конструктор квизов, который экономит 3 часа в неделю. Создайте квиз за 3 минуты с помощью AI, раздайте на планшеты, получите аналитику по каждому ученику.

**VP2 (для завуча/директора):**

> КвизОК — платформа интерактивного контроля знаний. Единая аналитика по всем учителям, классам и предметам. Работает офлайн на планшетах. Данные — в России.

**VP3 (для EdTech-бизнеса):**

> КвизОК — встраиваемая квиз-платформа для образовательных сетей. White Label, API, мультитенантность. Офлайн-first архитектура для регионов с плохим интернетом.

### 5.2. Офферы по тарифам

**Оффер MVP (Учитель-энтузиаст):**

> Бесплатно, навсегда. 5 квизов, 30 учеников, базовая аналитика, AI-генерация. Создайте первый квиз за 3 минуты — без установки, прямо в браузере.

**Оффер «Школа»:**

> ₽X/мес за всю школу. Безлимит учителей и квизов. Роли для завуча и директора. PDF-отчёты по классам. Приоритетная поддержка + обучение за 1 час.

**Оффер «Сеть учреждений»:**

> Индивидуальное предложение. Мультишкольная панель, кастомный бренд, API-интеграция с вашей LMS, персональный менеджер, SLA 99.9%, обучение педагогов.

### 5.3. УТП (сложно скопировать)

**УТП 1: Офлайн-first на уровне архитектуры**

> Квизы работают без интернета — не как «кэш», а как полноценное приложение с локальной базой, синхронизацией и queuing. Ни один конкурент (Kahoot, Quizziz, Plickers) не работает в школе без Wi-Fi.

**УТП 2: 8 типов вопросов с геймификацией**

> Не просто «A/B/C/D». Сопоставление, порядок, горячие точки на карте, заполнение пропусков — плюс серии, множители, жизни, звуки, хаптика. Это ближе к игре, чем к тесту.

**УТП 3: Аналитика, встроенная в педагогику**

> Автоматические инсайты: «Класс не понимает тему X», «Вопрос 5 слишком сложный», «Ученик Y сдал впервые за месяц». Учитель не анализирует данные — система делает это за него.

> **Head of Sales:** B2B-продажи в EdTech — это документы, не фичи. КП, оферта, кейс-стади — вот мой минимальный стек. Сейчас мне нечего отправить директору.

> **Design Director:** Визуально — 7/10, что хорошо для MVP. Но бренд рассыпается: «Konstruktor», «КвизОК», «Конструктор». Надо собрать в одно. Логотип уже есть — хорошо. Осталось сделать гайдлайн.

> **Методист:** Шаблоны по предметам — отличный ход. Но нужно маркировать по ФГОС и добавить классы (5–11). Методические рекомендации «Как использовать квизы на уроке» — обязательно.

> **CFO-lite:** При цене школьной лицензии ₽3–5k/мес и LTV 12 мес, нужно 20 школ для выхода на ₽1M ARR. Юнит-экономика рабочая, если CAC < ₽15k (1 визит в школу + 1 пилот). Бесплатный тариф — отличная воронка.
