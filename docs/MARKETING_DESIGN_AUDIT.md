## 1. Резюме

### Что мы продаём

Konstruktor — это **экосистема из трёх продуктов**:

| Продукт                   | Для кого        | Платформа                  | Назначение                          |
| ------------------------- | --------------- | -------------------------- | ----------------------------------- |
| **Конструктор** (Desktop) | Учителя         | Windows (Tauri)            | Создание квизов с ИИ, drag-and-drop |
| **Плеер** (Player)        | Ученики         | Android-планшеты + Windows | Прохождение квизов с геймификацией  |
| **CRM**                   | Учителя, завучи | Веб-браузер                | Аналитика результатов               |

### Кого убеждаем

| Роль                | Что хочет услышать                                     | Что хочет увидеть                                   |
| ------------------- | ------------------------------------------------------ | --------------------------------------------------- |
| **Директор школы**  | «Это повысит качество образования и упростит контроль» | Аналитику, отчёты, данные о прогрессе учеников      |
| **Завуч**           | «Учителя справятся сами, без IT-поддержки»             | Простой интерфейс, обучение за 5 минут              |
| **Учитель**         | «Мои ученики будут в восторге, а мне станет легче»     | ИИ-генерацию, красивый плеер, мгновенные результаты |
| **Учитель-скептик** | «Это не очередная игрушка?»                            | 8 типов вопросов, таймер, серьёзную аналитику       |

## 2. Оценка по продуктам

### 2.1 Конструктор (Desktop) — ⭐⭐⭐⭐ из 5

#### Что работает отлично

- **Splash-screen**: пружинная анимация логотипа, градиент indigo→purple, подпись «Квизы, которые запоминаются». Профессиональное первое впечатление.
- **AI-генерация**: ввёл тему → получил готовый квиз за 5 секунд. Основная «магия» продукта.
- **Редактор**: 8 типов вопросов (single, multiple, true/false, text, matching, ordering, fill_blank, hotspot). Drag-and-drop вопросов. Визуально чистый интерфейс.
- **Превью квиза**: полноценный Kahoot-стиль с таймером, плитками ответов, градиентными фонами по предметам.
- **Skeleton-загрузчики**: Профессиональные пульсирующие скелетоны вместо спиннеров.
- **Тексты 100% на русском**: Профессиональный, дружелюбный тон.

#### Проблемы для демо

| Проблема                                                                                                 | Влияние на ЛПР                                   | Серьёзность |
| -------------------------------------------------------------------------------------------------------- | ------------------------------------------------ | ----------- |
| **Сайдбар: 4 из 7 пунктов — заглушки** («Ученики», «Результаты», «Библиотека», «Шаблоны» ведут в никуда) | Директор кликнет и увидит пустоту — недоверие    | 🔴 Критично |
| **«Загрузить файл» и «Шаблон» — показывают тост «скоро»**                                                | Учитель разочаруется: обещали, но не работает    | 🔴 Критично |
| **Нет onboarding-тура** (Joyride подключён, но не активирован)                                           | Новый пользователь потеряется                    | 🟡 Высоко   |
| **Нет «Дублировать квиз»**                                                                               | Учителю приходится пересоздавать с нуля          | 🟡 Средне   |
| **Нет Undo/Redo**                                                                                        | При случайном удалении вопроса — данные потеряны | 🟡 Средне   |
| **Настройки: «Bearer токен», «API URL»** — технический жаргон                                            | Учитель без IT-специалиста не разберётся         | 🟡 Высоко   |
| **AI-генерация при первом открытии запускается автоматически** (если есть URL-параметры)                 | Может напугать непониманием                      | 🟠 Низко    |

### 2.2 Плеер (Player) — ⭐⭐⭐⭐½ из 5

#### Что работает отлично

- **Онбординг**: 4-шаговый мастер (имя → аватар из 8 эмодзи-животных → код учителя → загрузка). Дружелюбный, понятный ребёнку.
- **Countdown 3-2-1**: гигантская цифра с градиентом, звук обратного отсчёта. Создаёт «ритуал начала».
- **Звуковой дизайн**: 8 процедурных звуков (click, ding, buzz, streak-арпеджио, фанфара победы). Не файлы, а синтез — ~0 КБ.
- **Haptic feedback**: вибрация на Android при выборе ответа, правильном/неправильном ответе.
- **Streak-система**: badge «🔥 5 подряд» с Lottie-анимацией огня при streak ≥ 5, звёзд при ≥ 10.
- **Конфетти при score ≥ 60%**: canvas-confetti с 3 волнами (центр, левый угол, правый угол). Три секунды конфетти + фанфара = «вау».
- **Shareable результат**: canvas-генерация карточки 1080×1080 с аватаром, очками, прогресс-кольцом. Share API на Android.
- **Достижения**: «Идеальный результат», «Быстрый ответ», «Серия из 10».
- **Offline-first**: квизы скачиваются на устройство, работает без интернета.
- **Slide-переходы между страницами**: Framer Motion AnimatePresence, slide X + fade.

#### Проблемы для демо

| Проблема                                                                     | Влияние                                      | Серьёзность |
| ---------------------------------------------------------------------------- | -------------------------------------------- | ----------- |
| **text_input скрыт** (комментарий `{/* text_input — требует клавиатуры */}`) | 1 из 8 типов вопросов не работает в Player   | 🟡 Средне   |
| **Кнопка «Назад» позволяет выйти** из квиза без предупреждения               | Ученик может случайно выйти, данные потеряны | 🟡 Средне   |
| **Нет защиты от повторного открытия**                                        | Ученик может пройти квиз бесконечно          | 🟠 Низко    |
| **Hotspot может быть неточным** на маленьких экранах (пальцем)               | Учитель увидит — «это неудобно»              | 🟡 Средне   |
| **Нет имени учителя** на экране квиза                                        | Теряется «учительская» принадлежность        | 🟠 Низко    |
| **HomePage показывает «Select a quiz to start» по-английски**                | Нарушение единого языка                      | 🟡 Средне   |

### 2.3 CRM (Веб-аналитика) — ⭐⭐⭐ из 5

#### Что работает отлично

- **AI-инсайты**: Автоматические аналитические карточки: «Отличный результат класса 📊», «Большой разброс баллов», «Вопрос 4 вызывает 67% ошибок — стоит повторить тему 💡». С эмодзи и цветовой кодировкой.
- **Sparkline**: 7-дневный мини-график попыток на дашборде.
- **Skeleton-лоадеры**: Во всех страницах.
- **Таблица с сортировкой**: React Table с колонками Ученик/Балл/Статус/Дата.
- **Детализация сессии**: Покарточный разбор каждого ответа с цветовым кодированием и временем ответа.
- **Навбар**: Код организации (копируется по клику), управление Desktop-токеном.
- **Page transitions**: AnimatePresence с fade + Y-сдвигом.

#### Проблемы для демо

| Проблема                                                                | Влияние на ЛПР                             | Серьёзность                |
| ----------------------------------------------------------------------- | ------------------------------------------ | -------------------------- |
| **Полностью read-only** — нельзя создать, отредактировать, удалить квиз | Директор спросит: «А где всё остальное?»   | 🔴 Критично                |
| **Нет управления учениками/классами**                                   | Завуч: «Как мне увидеть 5Б класс?»         | 🔴 Критично                |
| **Нет экспорта данных** (CSV, PDF, Excel)                               | Завуч: «Как я это покажу родителям?»       | 🔴 Критично                |
| **Нет пароля сброса / регистрации**                                     | Учитель застрянет на логине                | 🔴 Критично                |
| **Нет графиков** (Recharts установлен, но не используется)              | Упущена визуализация данных                | 🟡 Высоко                  |
| **Empty state «Нет квизов»** — тупик без CTA                            | Новый пользователь не поймёт что делать    | 🟡 Высоко                  |
| **Нет фильтрации/поиска** по квизам                                     | С 50+ квизами — невозможно найти нужный    | 🟡 Средне                  |
| **GET /sessions/:id без авторизации**                                   | Любой с ID видит чужие данные — уязвимость | 🔴 Критично (безопасность) |

### 2.4 API — ⭐⭐⭐⭐ из 5

#### Что реализовано хорошо

- 17 эндпоинтов, RESTful архитектура.
- Два механизма авторизации: Better Auth (CRM) + Bearer API-tokens (Desktop).
- Устройства регистрируются с Argon2id-хешированием токена.
- JSONB для вопросов + нормализованная таблица `question_answers` для аналитики.
- Zod-валидация на входе (через @hono/zod-validator).
- CORS с whitelist.

#### Проблемы

- **Нет пагинации** — все запросы возвращают все записи. С 10к сессий — будет тормозить.
- **GET /sessions/:id без авторизации** — серьёзная дыра.
- **Нет rate-limiting**.
- **API-токен не имеет expiry** — бессрочный доступ.

## 3. Сценарий демо: что видит ЛПР

### Демо для директора школы (15 минут)

#### Минута 0–2: Запуск Desktop

**Что происходит:** Splash с пружинной анимацией → Дашборд.

**Впечатление:** ✅ «Красиво, современно, не как школьный журнал»

**Риск:** Если квизов нет — пустой экран. Нужен демо-квиз.

#### Минута 2–5: Создание квиза с ИИ

**Что происходит:** Учитель вводит «7 класс, Физика, Сила тяжести» → нажимает «Придумать» → через 5 секунд появляется готовый квиз с 5 вопросами.

**Впечатление:** ✅✅✅ «Вау, ИИ за 5 секунд создал тест?!» — **главный козырь**.

**Риск:** Если нет AI-ключа в настройках — ничего не произойдёт. Модалка скажет «Ключ не задан» — моментальный облом.

#### Минута 5–8: Превью квиза

**Что происходит:** Нажатие «Предпросмотр» → countdown 3-2-1 → Kahoot-стиль с градиентами, таймером, плитками.

**Впечатление:** ✅✅ «Как Kahoot, но своё и офлайн»

**Риск:** Без звука на десктопе впечатление слабее. Нет конфетти в Preview (только в Player).

#### Минута 8–10: Показываем Player на планшете

**Что происходит:** Ученик вводит имя, код учителя → скачивает квиз → проходит → конфетти → карточка результата.

**Впечатление:** ✅✅✅ «Дети будут в восторге» — звук, вибрация, confetti, streak.

**Риск:** Нужен настроенный планшет с APK. Если API недоступен — квиз не скачается (но если уже скачан — работает офлайн).

#### Минута 10–13: Показываем CRM

**Что происходит:** Открываем браузер → логин → дашборд → результаты квиза → AI-инсайты → карточка ученика.

**Впечатление:** ✅ «Вижу данные, понимаю кто не справился»

**Риск:** Если мало данных — инсайты не появятся. Если директор попросит «покажите экспорт в Excel» — его нет.

#### Минута 13–15: Вопросы ЛПР

| Вероятный вопрос                             | Текущий ответ       | Проблема                            |
| -------------------------------------------- | ------------------- | ----------------------------------- |
| «Можно ли загрузить тест из Word?»           | «Скоро»             | 🔴 Обещание без даты                |
| «Есть ли шаблоны?»                           | «Скоро»             | 🔴 Обещание без даты                |
| «Как ученики регистрируются?»                | «По коду»           | ✅ Но нет наглядной схемы           |
| «Можно экспортировать отчёт?»                | «Нет»               | 🔴 Для родительского собрания нужно |
| «Работает без интернета?»                    | «Да» (Player)       | ✅ Козырь                           |
| «Сколько стоит?»                             | Нет ценообразования | 🟡 Нужна pricing-страница           |
| «Есть ли интеграция с электронным журналом?» | «Нет»               | 🟡 На будущее                       |
| «А если у учителя Mac?»                      | «Windows only»      | 🟡 Ограничение платформы            |

## 4. Критические проблемы (блокеры продажи)

### 🔴 Блокер #1: Сломанная навигация Desktop

**4 из 7 пунктов сайдбара — заглушки.** «Ученики», «Результаты», «Библиотека», «Шаблоны» ведут в `#`. При демо директор кликнет → ничего не произойдёт → подорвано доверие.

**Решение:** Либо скрыть нереализованные пункты, либо показать красивый «coming soon» экран с иллюстрацией и датой.

### 🔴 Блокер #2: CRM без управления

CRM полностью read-only. Нельзя создать квиз, управлять учениками, экспортировать данные. Для завуча это не инструмент, а «витрина».

**Решение:** Добавить хотя бы экспорт CSV + базовый список учеников.

### 🔴 Блокер #3: Нет регистрации учителя

На LoginPage нет кнопки «Зарегистрироваться». Учитель не может сам создать аккаунт.

**Решение:** Добавить SignUp форму (Better Auth поддерживает email+password sign-up из коробки).

### 🔴 Блокер #4: Settings — технический жаргон

«Bearer токен», «API URL», «API Key» — учитель без IT-специалиста не понимает. В демо это выглядит как «для разработчиков, не для школы».

**Решение:** Переименовать: «Подключение к школьному серверу» вместо «API URL». Добавить кнопку «Проверить соединение». Или убрать из интерфейса и автоматизировать через QR-код.

### 🔴 Блокер #5: Безопасность API

`GET /sessions/:id` доступен без авторизации — любой с GUID видит результаты ученика.

**Решение:** Добавить `authMiddleware` на этот эндпоинт + проверку принадлежности к организации.

## 5. Что уже отлично (козыри при демо)

### 🏆 Козырь #1: ИИ-генерация квизов за 5 секунд

Это главный продающий аргумент. Ни Kahoot, ни Google Forms, ни Quizziz не делают ИИ-генерацию из коробки. Учитель вводит «5 класс, Литература, Пушкин» → получает готовый тест. **Это «вау-момент №1».**

### 🏆 Козырь #2: Офлайн-работа Player

Квизы скачиваются на планшет и работают без интернета. Для школ с плохим Wi-Fi — решающий аргумент. Синхронизация результатов происходит при появлении сети.

### 🏆 Козырь #3: Геймификация Player

Countdown 3-2-1, stagger-появление вариантов, звуковые эффекты, вибрация, streak-значки с Lottie-огнём, конфетти, shareable-карточка 1080×1080. Ученику хочется играть снова.

### 🏆 Козырь #4: AI-инсайты в CRM

Автоматический анализ: «67% ошибаются в вопросе 4 — стоит повторить тему». «Лучший результат: Иванов Петя — 96%». Учителю не нужно считать вручную.

### 🏆 Козырь #5: 8 типов вопросов

Не только single/multiple choice. Matching (соединение), ordering (сортировка), fill_blank, hotspot (точка на изображении). Гибкость = серьёзный инструмент, а не «простая игрушка».

### 🏆 Козырь #6: Shareable результат

Ученик может поделиться красивой карточкой результата (1080×1080 PNG) через соцсети. Бесплатный маркетинг от учеников.
