Konstruktor — Аудит v2: Продукт, который хотят купить
Философия: почему инструменты не продаются
Инструменты продаются по фичам. Продукты — по ощущениям. Excel — инструмент. Notion — продукт. PowerPoint — инструмент. Canva — продукт. Разница не в функционале — разница в том, что чувствует человек когда открывает.

Учитель открывает Konstruktor. У него 7 минут до урока. Он уставший. Ему не нужен «билдер с drag-and-drop». Ему нужно чувство: «я крутой учитель, потому что мои ученики в восторге от моих уроков». Вот что мы продаём — не квиз-конструктор, а ощущение суперсилы.

I. ОБЪЕДИНЕНИЕ КОНСТРУКТОРА И CRM — ЕДИНАЯ ВСЕЛЕННАЯ

Ученики проходят в Player

Три контекста. Три интерфейса. Три логина. Три стиля. Для директора школы — «а что, это разные программы?». Для учителя — лишние шаги.

Одно окно. Три режима.

Левый сайдбар — навигация:

📝 Мои квизы — список + создание
📊 Результаты — аналитика по квизам, ученикам, классам

III. ЧУВСТВЕННЫЙ ДИЗАЙН — ОТ FIGMA К BEHANCE
3.1 Визуальная система
Текущее: Tailwind дефолты. Slate серый. Blue-600 кнопки. Выглядит как admin panel из 2022 года.

Целевое: Тёплый, дружелюбный, немного игровой. Не «серьёзный софт для школы», а «магический инструмент для суперучителя».

Цветовая палитра:

Primary: Indigo-500 (#6366f1) — доверие + креативность
Accent: Violet-500 (#8b5cf6) — для ИИ-функций, «магия»
Success: Emerald-500 — не зелёный-светофор, а тёплый изумрудный
Background: Slate-50 (#f8fafc) для chrome, но с тёплым оттенком — добавить 1% бежевого через custom Tailwind color bg-warm-50: #faf9f7
Тексты: Slate-800 (не 700, не 900 — золотая середина)
Типографика:

Заголовки: Inter 700–900, letter-spacing: -0.02em — создаёт ощущение «современного продукта»
Body: Inter 400–500, line-height 1.6
Mono/numbers: JetBrains Mono — для скоров, таймеров, кодов
Тени и глубина:
Не плоский flat design, но и не material skeuomorphism. Нежная глубина:

Карточки: shadow-sm в покое → shadow-md на hover (с transition 200ms)
Модалки: shadow-2xl + backdrop blur 16px
Кнопки: shadow-sm в покое → при нажатии shadow-none + translate-y-[1px] (micro press effect)
Бордер-радиусы:
Увеличить везде. rounded-xl минимум для карточек, rounded-2xl для модалок, rounded-full для маленьких элементов. Острые углы = канцелярия. Скруглённые = дружелюбие.

3.2 Микро-анимации — то, что делает Apple Apple
Каждый элемент должен быть «живым». Не потому что анимации красивые, а потому что мёртвый интерфейс = страх ошибки, а живой интерфейс = уверенность.

Навигация между страницами:
Не жёсткая смена. Framer Motion <AnimatePresence>: текущая страница уходит влево с opacity 0 за 150ms, новая заезжает справа с opacity 1 за 200ms. Smooth, не навязчиво.

Список вопросов в сайдбаре:
При добавлении нового вопроса — он не просто «появляется». layout анимация Framer Motion: список раздвигается, новый элемент выезжает снизу с scale: 0.95 → 1, opacity: 0 → 1. При удалении — элемент сжимается и уходит. При перетаскивании — кинематический spring.

Кнопки:

Hover: scale: 1.02, transition: 150ms ease-out
Press: scale: 0.98, мгновенно
Primary кнопки: тонкий box-shadow: 0 0 0 4px rgba(99,102,241,0.15) на focus — halo-эффект
Переключение вкладок в модалках:
Индикатор под активной вкладкой — не просто бордер, а скользящий pill (layout анимация Framer Motion). Как в iOS segment control.

Toast-уведомления:
Вместо грубого красного/зелёного бара — элегантный toast, выезжающий снизу-справа: blur(12px) фон, иконка + текст, auto-dismiss через 3 секунды с плавным уходом вниз.

3.3 Иконография
Сейчас: эмодзи (🖼, 🎨, ✨, ❤️). Эмодзи — ок для прототипа, но для продукта:

Использовать Lucide Icons (уже стандарт для React-приложений): единообразный стиль, 24×24 SVG, 1.5px stroke. Эмодзи оставить только в контенте (вопросы/ответы), но НЕ в хроме приложения.

IV. QUIZ PLAYER — ОТ «ФОРМЫ» К «СОБЫТИЮ»
4.1 Вступительный экран квиза
Сейчас ученик выбирает квиз → сразу едет первый вопрос. Нет момента «приготовились».

Нужен countdown-экран:

Фон квиза на весь экран (тот самый AI-generated или цвет)
Название квиза крупно, предмет/класс
Имя учителя мелко
3-секундный обратный отсчёт: 3... 2... 1... 🎬
Каждая цифра — spring-анимация (масштабирование, затухание)
Фоновый звук готовности (тихий drum roll или мягкий whoosh)
Это создаёт ритуал начала. Дети ждут эти 3 секунды. Это как стартовый пистолет.

4.2 Вопросы — театральность подачи
Сейчас: Текст вопроса стоит наверху. Варианты стоят снизу. Статика.

Должно быть:

Вопрос выезжает сверху (slide down, 300ms, spring easing)
Варианты ответов появляются с задержкой — каждый следующий через 80ms (stagger animation). Это создаёт drumroll-эффект: «а вот и ответы...»
Таймер начинает тикать только после того как все варианты появились — fair play
4.3 Ответ — момент истины
Правильный ответ:

Кнопка: scale: 1.05 → зелёный цвет (300ms)
Burst-частицы из центра кнопки (8-12 маленьких кружков разлетаются во все стороны, fade out за 600ms). Чистый CSS: @keyframes burst { 0% { transform: scale(0); opacity: 1 } 100% { transform: scale(3) translateX(Npx); opacity: 0 } } на pseudo-elements
Звук: короткий высокий «ding» (200ms)
Вибрация: на Android-планшете — navigator.vibrate(100)
Streak-бейдж: если это 2+ правильных подряд — бейдж влетает сверху с bounce
Неправильный ответ:

Кнопка: shake-анимация (translateX: -8px → 8px → -4px → 4px → 0, 400ms)
Красный flash (100ms)
Правильный вариант подсвечивается зелёным (чтобы ученик видел и запоминал)
Звук: мягкий низкий «buzz» (150ms) — не злой, не обидный
Вибрация: navigator.vibrate([50, 30, 50]) — double tap
Если была жизнь — анимация ❤️ → 💔 (или сердце трескается)
4.4 Переходы между вопросами
Сейчас: мгновенная смена содержимого.

Должно быть: Текущий вопрос уходит влево (opacity fade + translateX -100%), новый заезжает справа. 250ms. Создаёт ощущение «пролистывания» — как в Stories или карусели. Каждый вопрос — новая «карточка».

4.5 Финишный экран — кульминация
Три сценария с разной энергией:

🏆 Отличный результат (≥90%):

Экран взрывается конфетти (canvas-based, 200 частиц, физика гравитации, 3 секунды). Библиотека canvas-confetti — 6KB.
Крупный «🏆» с golden glow анимацией
«Великолепно! 48 из 50 баллов» — текст появляется с typewriter-эффектом
Score counter: 0% → 96% за 1.5 секунды (ease-out кубическая)
✅ Хороший результат (≥60%):

Мягкое конфетти (50 частиц, 2 секунды)
«Отлично! Ты сдал!» — зелёный accent
Score counter стандартный
😤 Не сдал (<60%):

Без конфетти
«Попробуй ещё раз — ты сможешь!» — мотивирующий, не карательный
Кнопка «Повторить» с пульсирующим акцентом
На ВСЕ сценарии:

Секция «Разбор ошибок» — не сухая таблица, а карусель карточек: «Вопрос 3: ты ответил "Москва" — правильно было "Санкт-Петербург"». Swipeable на планшете.
Кнопка «Поделиться результатом» → генерация красивой карточки 1080×1080 (canvas → PNG → share). Дети делятся в соцсетях — бесплатный маркетинг.
V. КОНСТРУКТОР КВИЗОВ — ОТ EXCEL К MAGIC
5.1 ИИ — не фича, а ядро продукта
Сейчас ИИ-генерация — кнопка в углу. Это как если бы ChatGPT прятал поле ввода в подменю.

ИИ должен быть первым экраном при создании квиза:

Учитель нажимает «+ Новый квиз» → модальное окно:

Верхняя половина: «✨ Создать с ИИ» — hero-блок с анимированным градиентным фоном (purple shimmer)
Одно поле: «О чём квиз?» (большое, как поисковая строка Google)
Под ним: smart suggestions — «7 класс, Физика, Сила тяжести», «5 класс, Литература, Пушкин»
Кнопка «Создать» — при нажатии skeleton loading с пульсирующими серыми блоками (как LinkedIn/Facebook при загрузке контента). Через 5-8 секунд — готовый заполненный квиз «материализуется» в редакторе.
Нижняя половина: «Или создать вручную» — мелко, для опытных
Интеллектуальные подсказки в редакторе:
Учитель печатает текст вопроса → под полем ввода мягко появляется «✨ Предложить варианты ответов?» — ИИ допишет options. Это ghost-text паттерн (как Copilot в VSCode), только для учителей.

5.2 Шаблоны и пресеты
Сейчас: создание всегда с нуля.

Библиотека шаблонов:

«📚 Словарный диктант» — 20 matching-вопросов
«🧮 Устный счёт» — single_choice с числами
«📖 Проверка по главе» — mixed types
«❓ Блиц-опрос» — 10 true_false с 5-секундным таймером
Шаблоны — это не технический функционал. Это маркетинговый инструмент: учитель видит что продукт понимает его работу.

5.3 Предпросмотр в реальном времени
Сейчас учитель редактирует вопрос → жмёт «Превью квиза» → видит весь квиз. Это длинный feedback loop.

Split-view предпросмотр:
Справа от редактора — маленький «телефон» (мокап планшета, 200×280px) который в реальном времени показывает как текущий вопрос выглядит на ученическом устройстве. При наборе текста — текст меняется в мокапе live. Это как Webflow — edit + preview одновременно.

Вместо таблицы — аналитическая история:

Блок 1: Обзор (hero-секция)

Большой donut-chart: Сдали (зелёный) / Не сдали (красный)

Три метрики: Попыток / Ср. балл / Медианное время
Тёплый, приятный фон — может быть тот же AI-фон квиза, размытый
Блок 2: Распределение баллов

Histogram (bar chart): 0–20%, 20–40%, 40–60%, 60–80%, 80–100%
Каждый бар — цветовой градиент от красного к зелёному
Hoverable: при наведении — «12 учеников набрали 60–80%»
Блок 3: Проблемные вопросы

Отсортировано по % ошибок, от самого сложного к простому
Каждый вопрос: текст + процент ошибок + самый частый неправильный ответ
«💡 Рекомендация: 67% ошибаются в вопросе 4 — возможно, стоит объяснить тему повторно» — AI-generated insight (через gpt-5-nano, копеечная стоимость)
Блок 4: Таблица учеников

Сортируемая, с поиском
Sparkline прогресса ученика (если он проходил квиз несколько раз)
Click → детальная карточка ученика
6.3 Карточка ученика
Не просто «результат сессии», а профиль ученика:

Имя, класс, дата последней активности
Список пройденных квизов с баллами
Кривая прогресса (линейный график по датам)
«Сильные темы» / «Слабые темы» — если данных достаточно
Это то, что продаёт продукт завучу. Не «квиз-конструктор», а «система мониторинга знаний класса».

VII. ЗВУКОВОЙ ДИЗАЙН — НЕСЛЫШИМЫЙ ЭЛЕМЕНТ ВЛЮБЛЁННОСТИ
Почему TikTok залипает? Не только видео — звуки. Почему iPhone приятноеть? Тактильный click при наборе. Звук — 40% эмоционального воздействия, которое все игнорируют.

Набор звуков (8 файлов, суммарно ~200KB):

Звук Момент Характер
start.wav 3-2-1 countdown Три восходящих тона, как старт гонки
tick.wav Каждая секунда последних 5 Мягкий метроном, нарастающий
select.wav Выбор ответа Мягкий click (50ms)
correct.wav Правильный ответ Восходящий ding, мажор (200ms)
wrong.wav Неправильный ответ Мягкий buzz, не злой (150ms)

Критично: Кнопка «🔇 Без звука» на экране квиза — для классов где нужна тишина. Состояние сохраняется.

VIII. ТАКТИЛЬНАЯ ОБРАТНАЯ СВЯЗЬ (ANDROID ПЛАНШЕТЫ)
На сенсорных панелях 90% взаимодействия — пальцами. Haptic feedback — то, что делает физическую кнопку приятной:

Выбор ответа: navigator.vibrate(30) — лёгкий tap
Правильный ответ: navigator.vibrate(100) — удовлетворение
Неправильный: navigator.vibrate([40, 20, 40]) — двойной тук
Финиш: navigator.vibrate([100, 50, 100, 50, 200]) — победная дробь
Capacitor поддерживает @capacitor/haptics — более богатый API (impact light/medium/heavy). 5 минут на интеграцию.

IX. BRANDING — ИДЕНТИЧНОСТЬ ПРОДУКТА
9.1 Имя
«Konstruktor» — хорошее имя. Немного длинное, но запоминаемое. Русский продукт с немецким вайбом = надёжность + инженерия. Годится.

9.2 Логотип
Нужен logomark (знак без текста) + logotype (знак + текст).

Концепция знака: Буква K вписана в скруглённый квадрат (как иконка приложения). Левая вертикаль K — прямая, правая — расходится в две линии, напоминающие искру / звезду / magic wand — связь с AI-генерацией. Цвет: indigo-to-violet gradient.

Это будет работать как:

Иконка Tauri-приложения (taskbar, desktop)
Favicon CRM
Иконка Android APK
Splash-screen анимация
Watermark на карточках результатов
9.3 Tone of voice
Сейчас тексты — нейтрально-технические. «Введите текст вопроса», «Количество жизней».

Целевой тон — дружелюбный наставник:

Не «Ошибка: API ключ не задан» → а «Для работы ИИ нужен ключ — добавьте его в Настройках 😊»
Не «Фон не найден» → а «Фон ещё не выбран — создайте через ИИ или загрузите свой»
Не «No questions» → а «Пока пусто — добавьте первый вопрос!»
Пустые состояния — не тупик, а приглашение. Каждый empty state должен иметь иллюстрацию (simple line art, 200×160px) + подсказку с действием.

Streak counter + множитель. Достаточно для MVP, недостаточно для «влюблённости».

Аватарки учеников:
8 простых SVG-аватарок (как Duolingo owl, но проще — геометрические персонажи). Ученик выбирает при первом запуске. Аватарка отображается в результатах, в таблице учителя, на «подиуме» класса.

Leaderboard после квиза:
После того как все ученики прошли — учитель нажимает «Показать рейтинг» на интерактивной панели. На большом экране — анимированный подиум: 3 место → 2 место → 1 место (как в Mario Kart). С именами и аватарками. Дети кричат. Учитель — герой.

Система ачивок (долгосрок):

«Первый квиз» — 🌟
«5 подряд правильно» — ⚡
«Все вопросы правильно» — 🏆
«10 квизов пройдено» — 🎓
Ачивки — это не код на 3 дня. Это JSON-конфиг + рендер бейджей. Но они создают return value — ученик открывает Player снова, потому что хочет новую ачивку.
